# NNN. Short, descriptive title

- **Status:** Proposed | Accepted | Superseded by [NNN](./NNN-<slug>.md) | Deprecated
- **Date:** YYYY-MM-DD
- **Deciders:** name(s) of the people who agreed to this decision
- **Supersedes:** _(optional)_ link to a prior ADR this replaces

## Context

What is the situation that forced this decision? What constraints,
goals, or external pressures are at play? Keep this descriptive — do
not state the decision here.

## Decision

The single decision being recorded, in one or two short paragraphs.
State it as a present-tense fact, not as a recommendation:

> We adopt **X** for **Y**.

## Alternatives considered

For each alternative that was seriously evaluated, give a one-paragraph
sketch and the reason it was not chosen. The point is to make the
trade-off visible, not to write a full comparison matrix.

- **Alternative A** — short description. Rejected because …
- **Alternative B** — short description. Rejected because …

## Consequences

What changes because we made this decision? Cover both the upside and
the downside. Future maintainers should be able to read this section
and understand what they are signing up for.

- **Positive:** …
- **Negative:** …
- **Neutral / follow-ups:** anything that is implied but not yet done
  (links to issues, follow-up phases, etc.).

## References

- Links to related PRs, master-plan sections, external docs, prior
  discussion threads, etc.
